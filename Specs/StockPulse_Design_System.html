Design Principles

Core beliefs that guide every design decision in StockPulse.

1. Data Density Over Decoration

StockPulse is a power-user research tool. Every pixel should serve the data. Minimize whitespace waste, avoid decorative elements, and pack relevant information tightly. Think Bloomberg Terminal, not Instagram.

2. Scannable at a Glance

Users need to quickly assess a stock's health. Use color coding (green/red), consistent number formatting, and visual hierarchy to make key metrics instantly parseable without reading.

3. Predictable & Consistent

Every ticker symbol is clickable. Every table is sortable. Green means up, red means down. Once a user learns one page, they know them all. Zero surprises.

4. Functional, Not Fancy

No animations, no parallax, no gradients. This runs on localhost for one user. Speed and clarity beat aesthetics. Use a classless CSS framework (PicoCSS) and override minimally.

Number Formatting Rules

Prices: Always 2 decimal places → $142.56, never $142.6

Percentages: 1 decimal place → +4.2%, never +4.20%

Large numbers: Abbreviate with suffix → $3.49T, $706B, 52.3M

Ratios: 1 decimal place → P/E 55.3, Beta 1.67

Sign prefix: Always show + for positive changes → +$1,234, +0.98%

N/A: Use "N/A" (not "—" or blank) for missing numeric data

Responsive Behavior

Since this is a local desktop tool, responsive design is minimal. However, the layout should remain usable at viewport widths from 1024px to 1920px. Below 1024px, the 6-column metric grid collapses to 3 columns, and the sidebar layout stacks vertically.

Chart Standards

Library: Plotly.js (JSON specs generated by Python backend)

Color scheme: Navy (#1B2A4A) for primary line, Blue (#1976D2) for SMA50, Red (#C62828) for SMA200, Gray (#BDBDBD80) for volume bars

Background: White with light gray gridlines (#EEEEEE)

Period selector: 1M | 3M | 6M | 1Y | 5Y — default to 1Y

Hover: Show OHLCV tooltip with date, prices, and volume

Allocation charts: Use the brand color sequence: Navy, Blue, Green, Amber, Red, Purple, then Gray shades for overflow

Implementation Guidelines

Rules and conventions for building the frontend.

CSS Approach

Start with PicoCSS (classless CSS framework) as the base. Override with a single custom.css file. Target total custom CSS at under 500 lines.

✓ Do

Use semantic HTML elements (<table>, <nav>, <article>)

Rely on PicoCSS defaults for base styling

Use CSS custom properties for all colors

Use CSS Grid for page layouts

Keep one CSS file (custom.css)

✗ Don't

Add Tailwind, Bootstrap, or other large frameworks

Use inline styles in templates

Create component-specific CSS files

Use CSS-in-JS or CSS modules

Add CSS animations or transitions (except hover color changes)

JavaScript Approach

✓ Do

Use HTMX for all server interactions (forms, tabs, pagination)

Use Alpine.js for small client-side state (dropdowns, modals, toggles)

Use Plotly.js for all charts (fetch JSON specs from API)

Keep JS in one app.js file for keyboard shortcuts + global helpers

✗ Don't

Use React, Vue, or any SPA framework

Write custom AJAX/fetch calls (use HTMX)

Build client-side routing

Add build tools (Webpack, Vite, esbuild)

Use npm/node for frontend dependencies (load from CDN)

Template Conventions (Jinja2)

Base template: base.html contains nav, head (CSS/JS imports), footer, and defines blocks: {% block title %}, {% block content %}

Full pages: Extend base.html. Located in templates/ root. Named by route: dashboard.html, screener.html.

HTMX partials: Stand-alone HTML fragments (no base extend). Located in templates/partials/. Named by function: ticker_financials.html, portfolio_table.html.

Naming: snake_case for all template files. Match router function names.

Accessibility Baseline

Since this is a personal tool, full WCAG compliance isn't required, but these basics make the tool more usable:

All interactive elements must be keyboard-focusable. Tables must have <thead> with <th> elements. Color is never the only indicator — always pair with + / - signs or text labels. Form inputs must have associated <label> elements.
